POST Login

Request

Field Type Required Description Values
subCustomer string Yes Defines the name of the subclient configured on the platform Text string

session

idSession string The web session the user is in Text string
timeSession integer($int32) The timestamp when the session was issued in Unix epoch format Text string
channel string User channel, web or mobile Text string

login

user

user_id string A unique identifier of the user, as given by the app Text string
email string User email Text string
userFullName string User's full name Text string
userName string User name Text string
checkpoint_type string Yes The type of checkpoint to invoke Text string

webParams

checkpoint string ndicates the request's relevant checkpoint 'LOGIN', 'REGISTER', 'TRANSACTION' or 'ADD_PAYMENT_METHOD' Supported Checkpoints list will be deliverd upon integration.
version number($double) The API version invoked
user_id string User identification number. Text string

Example

  • curl -X POST \
  • 'https://sandbox.idfactory.me/TransactionFraud/api/Login' \
  • -H 'accept: application/json' \
  • -H 'Authorization: Bearer AKFGBAERUIPG65465' \
  • -H 'Content-Type: application/json' \
  • -d '{
  • "subCustomer": "string",
  • "session": {
    • "idSession": "string",
    • "timeSession": 0,
    • "channel": "string"
  • },
  • "login": {
    • "user": {
      • "user_id": "string",
      • "email": "string",
      • "userFullName": "string",
      • "userName": "string"
      • },
    • "checkpoint_type": "string"
    • },
  • "webParams": {
    • "checkpoint": "string",
    • "version": 0,
    • "user_id": "string"
    • }
  • }'

Note:

  • Replace for name of the sub customer in the "SubCustomer" field.
  • Replace for your id number in the "identificationNumber" field
  • Replace for id of identification type in the "identificationType" field
